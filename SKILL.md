# مهارة الإصدار التلقائي للرفع على GitHub Releases

هذه المهارة مصممة لتكون عامة وقابلة للاستخدام في **مشاريع متعددة ومختلفة** (مهما كانت لغة البرمجة). عند استدعائها، ستقوم بصفتك مساعد الذكاء الاصطناعي بتنفيذ سلسلة من المهام التلقائية لرفع التحديثات وإصدار نسخة جديدة من المشروع.

## متى تُستخدم هذه المهارة؟
استخدم هذه المهارة عندما يطلب منك المستخدم: "ارفع الإصدار"، "عمل إصدار جديد"، "نشر التحديثات"، "Release"، أو أي طلب مشابه يهدف لإنهاء التحديثات ورفعها.

---

## 🛠️ الخطوات الآلية التي يجب عليك (كذكاء اصطناعي) تنفيذها بالترتيب:

بدلاً من تنفيذ أوامر Git يدوياً، قم بتشغيل سكربت البايثون المرفق `auto_release.py` والذي سيقوم بتولي جميع المهام (تحديد الإصدار، تحديث ملف VERSION، الـ Commit، الـ Push، رفع الـ Tag، وإنشاء الـ GitHub Release).

1. تحليل التغييرات لتحديد الإصدار وكتابة التعليق تلقائياً:
   - قم بمراجعة التعديلات غير المحفوظة (سواء من سياق المحادثة والتعديلات التي قمت بها، أو عبر قراءة `git status` و `git diff`).
   - استنتج **تلقائياً** نوع التحديث المناسب (لا تسأل المستخدم بل قرر بناءً على حجم العمل):
     - **patch**: إذا كانت التعديلات مجرد إصلاح أخطاء (Bug fixes) أو تحسينات صغيرة.
     - **minor**: إذا قمت بإضافة ميزات أو شاشات جديدة.
     - **major**: إذا قمت بتغييرات ضخمة أو إعادة هيكلة شاملة.
   - قم بصياغة "ملخص التحديثات" (Commit Message) بشكل دقيق يعكس ما تم إنجازه فعلياً.

2. اطلب من الطرفية (Terminal) تشغيل السكربت بالصيغة التالية مع استبدال المتغيرات:
   ```bash
   python auto_release.py --type <نوع_التحديث> --message "<ملخص_التحديثات>"
   ```
   **مثال عملي:**
   `python auto_release.py --type patch --message "إصلاح بعض الأخطاء البرمجية وإضافة دعم اللغة العربية"`

3. بعد الانتهاء، اقرأ مخرجات السكربت وأخبر المستخدم بالنتيجة النهائية ورقم الإصدار الجديد الذي تم إطلاقه بنجاح!

---
## ⚠️ ملاحظات هامة:
- لا تطلب من المستخدم تنفيذ الأمر، بل **قم بتنفيذه بنفسك** عبر أداة `run_command` في الطرفية.
- تأكد من وجود ملف `auto_release.py` في المسار قبل التشغيل.
