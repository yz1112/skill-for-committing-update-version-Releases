# أتمتة الإصدارات وسير عمل Git (Auto Version Release Skill) 🚀

هذا المستودع يحتوي على مهارة (Skill) مخصصة لمساعد الذكاء الاصطناعي. تهدف هذه المهارة إلى أتمتة دورة حياة الإصدارات (Versioning) وعمليات الـ Commit والـ Push والـ Release على منصة GitHub في أي مشروع، مهما كانت لغة البرمجة أو إطار العمل المستخدم.

## 🎯 الهدف من المهارة
- **توفير الوقت والجهد:** بدلاً من إدخال أوامر Git المتعددة يدوياً وحفظ أرقام الإصدارات وتحديثها، يقوم الذكاء الاصطناعي نيابة عنك بذلك.
- **التوحيد القياسي:** ضمان استخدام سير عمل (Git Workflow) موحد ومستقر لجميع مشاريعك المختلفة.
- **وضوح الإصدارات محلياً:** تقوم المهارة بإنشاء وتحديث ملف `VERSION` تلقائياً في الجذر الرئيسي للمشروع ليكون رقم الإصدار الحالي واضحاً لجميع المطورين وفي أي وقت.
- **إدارة احترافية للتحديثات:** أتمتة عملية إنشاء `Git Tag` و `GitHub Release` مع ملاحظات التحديث (Release notes) بشكل آلي واحترافي.

## ⚙️ كيف تعمل المهارة؟

بمجرد استدعاء المهارة، سيقوم الذكاء الاصطناعي تلقائياً بتشغيل سكربت البايثون المرفق `auto_release.py` والذي يقوم خلف الكواليس بـ:
1. **الاستعلام عن الإصدار الحالي:** من خلال Tags السابقة أو ملفات المشروع (مثل `VERSION`).
2. **تحديد الإصدار الجديد:** بناءً على حجم التعديلات وتطبيق مفهوم (Semantic Versioning: Patch, Minor, Major).
3. **توضيح الإصدار محلياً:** تحديث أو إنشاء ملف `VERSION` بالرقم الجديد.
4. **أتمتة الحفظ والرفع:** تنفيذ `git add`، و `git commit` برسالة واضحة، ثم `git push`.
5. **إنشاء الإصدار (Release):** عمل `git tag` ورفعه، ثم استدعاء أوامر `gh` لإنشاء الـ Release في صفحة المشروع.

## 🛠️ كيفية الاستخدام والتركيب

تم تجهيز هذا المستودع ليكون متوافقاً وقابلاً للقراءة من قبل أشهر مساعدات الذكاء الاصطناعي مباشرة (Plug & Play). كل ما عليك فعله لتفعيل المهارة في أي مشروع جديد هو:

1. **نسخ المهارة الأساسية:**
   - انسخ ملف `SKILL.md` وضعه في المسار الرئيسي لمشروعك.

2. **نسخ ملفات التفعيل (بحسب بيئة عملك):**
   انسخ الملف المناسب لمحررك لكي يقوم الذكاء الاصطناعي بالتعرف على المهارة تلقائياً:
   - **Cursor AI:** انسخ ملف `.cursorrules`
   - **Claude Code (Anthropic):** انسخ ملف `CLAUDE.md`
   - **Windsurf / Codeium:** انسخ ملف `.windsurfrules`
   - **GitHub Copilot / VSCode Agents:** انسخ المجلد `.github` كاملاً (الذي يحتوي على `copilot-instructions.md`)
   - **Antigravity (Gemini):** سيتعرف على المهارة بمجرد وجود `SKILL.md` ضمن سياق المشروع أو يمكنك الطلب منه قراءتها.

*(تضمن هذه الملفات أن الذكاء الاصطناعي سيقوم بقراءة وتنفيذ مهارة `SKILL.md` بكل دقة بمجرد أن تطلب منه ذلك).*
2. **التنفيذ:**
   عندما تنتهي من التعديلات البرمجية في مشروعك وتكون جاهزاً لنشر إصدار جديد، قم بتوجيه أمر بسيط للذكاء الاصطناعي باستخدام لغتك الطبيعية، مثل:
   - *"ارفع الإصدار"*
   - *"اعمل إصدار جديد"*
   - *"ارفع التحديثات ونفذ مهارة الإصدار"*
   - *"Release this version"*

سيقوم المساعد تلقائياً بمعرفة ما يجب فعله وتنفيذ سلسلة الأوامر وراء الكواليس وتقديم تقرير مبسط بالنتيجة.

## ⚠️ المتطلبات المسبقة (Prerequisites)
- [Git](https://git-scm.com/) مثبت على جهازك والمشروع مربوط بمستودع (Repository).
- أداة [GitHub CLI (`gh`)](https://cli.github.com/) (يُفضل بشدة) مثبتة ومسجلة الدخول على جهازك، وذلك لتمكين الذكاء الاصطناعي من إنشاء الـ Release مباشرة من الطرفية. في حال لم تكن الأداة موجودة، ستكتفي المهارة برفع الـ Tag وسيتوجب عليك تحويله لـ Release يدوياً من الموقع.
